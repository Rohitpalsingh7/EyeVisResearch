The Problem

Sharing

Sharing eye-tracking data in a meaningful way is not easy; the leading manufacturers of eye tracking software lock down their visualizations to proprietary formats, which require expensive software to use and modify. This means that eye-tracking labs must find workarounds, or create reports and exports, which can lose or change meaning.

Manipulating

When working with exports and reports, the viewer loses the ability to manipulate the visualizations. Tracking exports are, typically, static formats such as images, which are rendered once.

Agility

Static data is hard to derive meaning from, or to juxtapose with other data. A rendered image can show a summary but not a narrative. It also exists only for the time in which it was rendered. Any new data updates or additions require an entirely new image to be created. This can be cumbersome and relies on keeping in contact with the technician that created the initial renders.

The Solution

Eyetracker.js

Eyetracker.js provides User Experience Designers and Technicians with the ability to share their data in an interactive way that is familiar to their users.

Gaze Plots

The tool creates familiar renderings such as the gaze plot, which allows for the viewer to see the major 'fixations' and 'saccades' for the dataset in a sequential order.

Heat Maps

Another standby in eye-tracking visualizations is the Heat Map, which shows areas of interest based on the frequency in which a viewer looked at them. Eyetracker.js renders the Heat map in real time and adds it right into the same display for comparison to the other visualization idioms.

Playback

The tool used the idiom of a video player to allow viewers to manipulate data in real time. In showing the data this way, clients whoa re familiar with streaming players like YouTube, Netflix, or Vimeo can easily interact with the data in a meaningful and intuitive way.
